# 调试44004错误指南

## 问题说明

你遇到了错误码44004（文本内容为空），即使使用简化格式也会报错。

## 调试步骤

### 步骤1：等待Docker镜像重新构建

访问GitHub Actions查看构建进度：
```
https://github.com/gowfqk/WeChat/actions
```

等待构建完成（通常需要3-5分钟）。

### 步骤2：拉取最新镜像

```bash
docker pull gowfqk/go-wecomchan:latest
```

### 步骤3：停止旧容器并启动新容器

```bash
# 停止旧容器
docker stop <容器名或容器ID>

# 启动新容器
docker run -d -p 8080:8080 \
  --name wecomchan \
  -e SENDKEY=set_a_sendkey \
  -e WECOM_CID=你的企业微信公司ID \
  -e WECOM_SECRET=你的企业微信应用Secret \
  -e WECOM_AID=你的企业微信应用ID \
  -e CACHE_TYPE=memory \
  gowfqk/go-wecomchan:latest
```

### 步骤4：查看容器日志

```bash
# 实时查看日志
docker logs -f wecomchan
```

### 步骤5：发送测试请求

在新终端中执行：

```bash
# 测试1：简化格式
curl -X POST http://localhost:8080/wecomchan \
  -H "Content-Type: application/json" \
  -d '{
    "msg_type": "text",
    "msg": "测试消息"
  }'
```

### 步骤6：分析日志输出

在`docker logs -f wecomchan`的输出中，你会看到：

```
========== 收到新请求 ==========
请求方法: POST
请求URL: /wecomchan
Content-Type: application/json
请求体内容: {"msg_type": "text", "msg": "测试消息"}
解析结果 - sendkey: '', msgType: 'text'
requestBody.Msg: '测试消息'
使用body传参（简化格式）
最终参数 - sendkey: '', msgType: 'text', msgContent: '测试消息'
环境变量Sendkey: 'set_a_sendkey'
sendkey验证失败 - 期望: 'set_a_sendkey', 实际: ''
```

## 可能的问题和解决方案

### 问题1：sendkey验证失败

**日志特征**：
```
sendkey验证失败 - 期望: 'xxx', 实际: ''
```

**解决方案**：

在请求中添加sendkey：

```bash
curl -X POST http://localhost:8080/wecomchan \
  -H "Content-Type: application/json" \
  -d '{
    "sendkey": "set_a_sendkey",
    "msg_type": "text",
    "msg": "测试消息"
  }'
```

或者设置环境变量：

```bash
docker run -d -p 8080:8080 \
  -e SENDKEY=set_a_sendkey \
  -e WECOM_CID=xxx \
  -e WECOM_SECRET=xxx \
  -e WECOM_AID=xxx \
  -e CACHE_TYPE=memory \
  gowfqk/go-wecomchan:latest
```

### 问题2：msgContent为空

**日志特征**：
```
错误：msgContent为空
```

**原因**：
- 请求体格式错误
- JSON解析失败
- 没有提供消息内容

**解决方案**：

检查请求格式是否正确：

```bash
# 正确格式
curl -X POST http://localhost:8080/wecomchan \
  -H "Content-Type: application/json" \
  -d '{
    "sendkey": "set_a_sendkey",
    "msg_type": "text",
    "msg": "测试消息"
  }'
```

### 问题3：企业微信API返回44004

**日志特征**：
```
准备发送的数据: {...}
消息内容长度: X
企业微信发送应用消息接口返回==>{errcode:44004, errmsg:text content is empty}
```

**原因**：
- 发送到企业微信的数据格式错误
- msgContent在传输过程中被清空
- 编码问题

**解决方案**：

检查`准备发送的数据`日志，确认msg字段的内容是否正确。

## 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| `invalid sendkey` | sendkey验证失败 | 在请求中添加sendkey或检查环境变量 |
| `text content is empty` | msgContent为空 | 检查请求格式和JSON语法 |
| `44004` | 企业微信API接收到的content为空 | 查看完整日志确认发送的数据 |
